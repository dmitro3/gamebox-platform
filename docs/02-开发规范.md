# 开发规范

## 命名
| 类型 | 规范 | 例 |
|------|------|----|
| 目录 | kebab-case | `game-lobby/` |
| Vue 组件 | PascalCase | `GameLobby.vue` |
| Nest 文件 | `xxx.controller/service/module.ts` | `points.service.ts` |
| 常量/枚举值 | 全大写下划线 | `PLATFORM_FEE_RATE` |

## 代码
- Prettier（根 `.prettierrc`）+ ESLint；提交前格式化（husky + lint-staged）。
- TS 严格模式，禁滥用 `any`。
- 金额一律整数（积分最小单位）。
- 注释只解释「为什么」。

## Git
- 分支：`main`(受保护) / `dev` / `feature/*` / `fix/*` / `release/*`。
- 提交：Conventional Commits `feat(scope): ...`，scope ∈ server/client-app/admin-panel/games/shared。

## API
- RESTful；JWT（`Authorization: Bearer`）。
- 统一响应：成功 `{code:0,data,message}`；失败 `{code,data:null,message}`（见 `response.interceptor` / `http-exception.filter`）。
- 分页 `?page=&pageSize=` → `{items,total,page,pageSize}`。
- 错误码段位：400xx 参数/业务、401xx 未登录、403xx 无权限、404xx 不存在、409xx 冲突、500xx 服务端。
- 接口契约类型统一进 `packages/shared`。

## 环境变量
- 每模块 `.env.example`（提交）/ `.env`（不提交）。
- 后端读取走 `@nestjs/config`，不散用 `process.env`。
- 密钥只进 `.env`/密钥管理，绝不入库。

## 安全红线（博彩类必守）
1. 开奖/爆率/积分只在 server；前端随机全删。
2. 写接口后端必再校验；账本走事务+幂等。
3. 数据库不公网；上线删后门（含原型测试账号）。
4. 可证明公平 + RTP 监控 + 风控告警。
5. 关键操作写 `AdminLog`。

## 提交前检查
- [ ] lint/类型通过 · 无 console 残留 · 无硬编码密钥
- [ ] 涉及积分走了账本+事务
- [ ] 新接口类型已进 shared
- [ ] commit 符合规范
