# 落地方案

> 总纲见 [`00-最优架构方案.md`](./00-最优架构方案.md)。本文聚焦目录、迁移与部署。

## 目录结构（项目根 = bjNew/bj）

```text
bjNew/bj/
├── package.json / pnpm-workspace.yaml / .gitignore / .editorconfig / .prettierrc
├── server/                      # 后端大脑（NestJS）★已搭骨架
│   ├── prisma/schema.prisma     # 数据库设计（就绪）
│   ├── prisma/seed.ts           # 种子（骨架）
│   ├── .env.example
│   └── src/
│       ├── main.ts / app.module.ts
│       ├── prisma/              # PrismaService
│       ├── common/             # 响应拦截器 / 异常过滤器
│       └── modules/
│           ├── game-core/      # 引擎抽象 IGameEngine + 注册表 + 可证明公平
│           ├── points/         # 积分账本（核心）
│           └── agents/         # 多级分润
├── packages/shared/             # 多端共享：enums / validators / format ★就绪
├── client-app/                  # 玩家/代理/代分/分公司端（现为原型，待迁 Vue3+Capacitor）
├── admin-panel/                 # 管理后台（现为原型，待迁 Vue3+ElementPlus）
├── games-assets/                # H5 游戏（多款原型，待接 server）
└── docs/                        # 本套文档
```

> 现有 `client-app / admin-panel / games-assets` 原生原型保留为**迁移蓝本**，按 `05-工程实施清单.md` 分阶段迁移。

## 原型逻辑去向
| 原型 | 去向 |
|------|------|
| 账号校验 / 金额格式化 | `packages/shared` |
| 积分/爆率/开奖/分润/审核（权威） | **server** |
| toast / 弹窗 / tabbar / 音效 | client-app 全局组件 |
| 页面交互 | client-app `views/*.vue` |
| localStorage / mock 数据 | client-app `services` + `stores`，改调 server |

## 阶段一部署（单机）
Nginx 统一入口：`/api`→server、`/ws`→WS、`/admin`→admin-panel、`/games`→games-assets、`/download`→APK 页；PG + Redis 同机。
约束：server 无状态、域名配置化 → 将来可平滑拆分/上 CDN。

## 开发顺序（里程碑）
M0 脚手架 → M1 后端地基(账本/鉴权) → M2 APP 连后端样板 → M3 出 APK → M4 资金+一款游戏闭环 → M5 后台 → M6 补全(多游戏/活动/客服) → M7 上线。详见 `05`。
