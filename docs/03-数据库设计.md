# 数据库设计

> 可直接使用的 schema 见 [`server/prisma/schema.prisma`](../server/prisma/schema.prisma)。本文是配套说明。

## 原则

1. server 唯一权威；2. 积分账本是变动唯一入口（流水+balanceAfter+幂等+事务）；
3. 金额用整数；4. 配置版本化；5. 可证明公平字段。

## 表清单

### 用户 / 多级代理树
| 表 | 职责 | 关键字段 |
|----|------|----------|
| `User` | 所有用户（5 角色） | `parentId` 自引用、`agentPath` 物化路径、`depth`、`promoCode`、`role` |
| `Agent` | 代理扩展 | `levelId`、`commissionRate`、`teamFlowTotal`、`pointsQuota` |
| `AgentLevel` | V1-V5 等级配置 | `minTeamFlow`、`commissionRate`（对应原型 team-data） |

> 角色树：ADMIN → BRANCH → AGENT → PROXY → PLAYER，用 `parentId/agentPath/depth` 表达，分润沿链逐级。

### 账本 / 资金
| 表 | 职责 |
|----|------|
| `PointsAccount` | 各主体积分账户（PLATFORM 可为负，余可冻结） |
| `PointsLedger` | 流水（amount 正入负出、balanceAfter、idempotencyKey、refType/refId） |
| `RechargeOrder` | 上下分申请审核 |

### 游戏 / 爆率 / 注单
| 表 | 职责 | 关键点 |
|----|------|--------|
| `Game` | 游戏目录 | `category`（4 类）、`drawIntervalSec`（彩票周期） |
| `GameConfig` | **爆率/赔率配置** | `payTable` JSON、版本化、`active` 唯一、前端拿不到 |
| `LotteryIssue` | 彩票期号 | `issueNo`、`status`、`openAt/lockAt`、`drawNumbers`、`serverSeedHash` |
| `GameRound` | 棋牌/电子/街机一局 | `category`、`outcome`、可证明公平字段 |
| `Bet` | **统一注单** | 彩票关联 `issueId`+`betType`+`odds`；棋牌/电子关联 `roundId`；`validFlow` 算分润 |

### 分润 / 回水
| 表 | 职责 |
|----|------|
| `CommissionRecord` | **多级分成**：`beneficiaryId`+`sourceUserId`+`level`，逐级一条 |
| `RebateRecord` | 玩家回水 |

### 活动 / 客服 / 运营 / 风控
| 表 | 职责 |
|----|------|
| `Activity` / `ActivityClaim` | 活动配置与领取 |
| `CsMessage` / `Feedback` | 客服消息 / 反馈 |
| `AppVersion` | APP 版本（下载/强更） |
| `RiskEvent` | 风控事件（RTP 偏离/大额/异常胜率/对刷） |
| `AdminLog` | 管理操作审计 |

## 原型 → 数据库映射
| 原型 localStorage / mock | 表 |
|--------------------------|----|
| 账号表 / 当前用户 | `User` |
| 余额 | `PointsAccount` + `PointsLedger` |
| team-data（等级/下线/佣金） | `AgentLevel` + `Agent` + `User`(树) + `CommissionRecord` |
| flow-data（流水） | `PointsLedger` / `Bet` |
| recharge_history | `RechargeOrder` |
| cs_messages / feedback | `CsMessage` / `Feedback` |

## 开发期
开发可临时把 `provider` 改 `sqlite` 跑通；生产 PostgreSQL（独立实例+备份）。
